Lecture 11: Time & Space Complexity Part 2

Problem Statement

Is lecture mein hum loop-based time complexity ko deeply analyze karenge. Har tarah ke loop pattern — exponential increment, nested loops, variable inner loops — ka exact derivation samjhenge. Saath hi Best, Average, Worst Case aur Big-O, Omega, Theta notations ka complete comparison karenge.

Intuition

Time complexity calculate karne ka sabse asaan tarika yeh hai ki hum loop ke andar kitni baar code execute ho raha hai usko count karein. Har iteration ka cost 1 maan ke total iterations ka sum nikaal lo. Uske baad sabse badi power wale term ko pakad lo — wahi T.C. hoti hai.

Example 4: Triple Nested Loop (O(n³))

Code Snippet

// Three nested loops, each running n times for (int i = 0; i < n; i++) { for (int j = 0; j < n; j++) { for (int k = 0; k < n; k++) { cout << "hello"; } } }

Complexity Analysis

Complexity Analysis

Outer loop n baar chalega. Har outer iteration ke andar middle loop n baar. Har middle iteration ke andar inner loop n baar.

Total iterations = n × n × n
T.C. = O(n³)

Example 5: Variable Inner Loop (Dependent on Outer)

Code Snippet

// Inner loop depends on outer loop variable i for (int i = 1; i < n; i++) { for (int j = 0; j < i * i; j++) { cout << "hello"; } }

Step-by-Step Dry Run

Dry Run / Visualization
Outer iInner j runsIterations
i = 1j = 0 to 01 time
i = 2j = 0 to 34 times
i = 3j = 0 to 89 times
i = 4j = 0 to 1516 times
.........
i = n-1j = 0 to (n-1)² - 1(n-1)² times

Mathematical Derivation

Sum = 1² + 2² + 3² + 4² + ... + (n-1)²
Formula: Σk² = n(n+1)(2n+1) / 6
For (n-1) terms: (n-1)(n)(2n-1) / 6
Expand: (2n³ - 3n² + n) / 6
Dominant term: 2n³ / 6 = n³ / 3
T.C. = O(n³)
Key Insight: "Sabse badi value ko pakad ke chalna" — jab bhi polynomial sum aaye, toh sabse highest degree wale term ko utha lo. Constants aur lower degree terms ko ignore kar do. Yahi Big-O analysis ka core principle hai.

Notation Deep Dive

O Big-O (Upper Bound)

Big-O notation sabse zyada time batata hai — matlab algorithm kabhi bhi isse zyada time nahi lega. Interview mein hamesha Big-O pucha jata hai kyunki yeh upper bound provide karta hai.

Example: Agar T.C. = n³ hai, toh Big-O = O(n³). Algorithm n³ se zyada time nahi lega.

Ω Omega (Lower Bound)

Omega notation sabse kam time batata hai — matlab algorithm itna time toh minimum lega hi. Yeh lower bound hota hai. Agar T.C. = n³ hai, toh Ω(n³) bhi hoga kyunki kam se kam itna time toh lagega.

Example: Ω(n³) ka matlab algorithm n³ se kam time mein complete nahi ho sakta.

Θ Theta (Tight Bound)

Theta tab use hota hai jab exact time complexity pata ho. Jab upper bound aur lower bound same hote hain, tab Theta use karte hain. Lekin practically, Theta sirf tab use karte hain jab T.C. exactly known ho.

Real World: Mostly Big-O hi use hota hai kyunki exact bound nikaalna mushkil hota hai.

Important

Teen notations — Big-O, Omega, Theta — mainly confusion ke liye rakhe gaye hain. Interview aur competitive programming mein sirf Big-O hi use karte hain. Kyunki Big-O mein hum hamesha upper bound consider karte hain, jo sabse safe estimate hota hai.

Interview Tips & Common Pitfalls

Interview Tip
  • Hamesha worst case complexity batana — interviewer ko yeh pasand hai.
  • Nested loops ke case mein multiply karo iterations ko, add nahi.
  • Exponential loops (×2, ×3, /2) ka base matter nahi karta class ke liye — sab O(log n) mein aate hain.
  • Sum of squares formula yaad rakho: Σk² = n(n+1)(2n+1)/6
  • Big-O, Omega, Theta mein sirf Big-O hi practically use hota hai.
Edge Case
  • Agar n = 0 ho toh loop kabhi execute nahi hoga — complexity technically O(1) (constant time check).
  • Integer overflow ka dhyan rakho: i = i * 2 mein agar i int hai toh overflow ho sakta hai bahut bade n ke liye.
  • while(i > 0) mein agar i negative ho jaye toh infinite loop — isliye unsigned ya proper check zaroori hai.

Key Takeaways

Takeaway
  1. Loop analysis ka basic rule: har iteration ka cost 1 maan ke total iterations count karo.
  2. Exponential increment/decrement loops (×2, ×3, /2) dete hain O(log n) complexity.
  3. Nested independent loops multiply hote hain: n × n × n = O(n³).
  4. Dependent inner loops mein series sum nikaal ke highest degree term pakdo.
  5. Sabse badi value ko pakad ke chalna — lower order terms aur constants ko ignore karo.
  6. Interview mein hamesha Worst Case + Big-O batana.
  7. Big-O = Upper Bound, Omega = Lower Bound, Theta = Tight Bound. Practically sirf Big-O use hota hai.
