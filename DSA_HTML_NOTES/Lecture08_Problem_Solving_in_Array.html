C++ Vector Fundamentals

Pattern Data Structure — Dynamic Array, STL Container, Random Access

Introduction

C++ mein vector ek dynamic array hai jo Standard Template Library (STL) ka part hai. Ye fixed-size arrays se different hai kyunki iska size runtime pe grow aur shrink ho sakta hai. Vector contiguous memory allocate karta hai, jisse random access O(1) time mein possible hota hai. Ye DSA problems solve karne ke liye sabse commonly used container hai.

Key Insight: Vector dynamic array hai jo apne aap resize hota hai. Jab capacity full hoti hai, ye double capacity allocate karta hai aur existing elements ko copy karta hai. Isliye push_back ka amortized time O(1) hota hai.

Syntax and Usage

Vector use karne ke liye <vector> header file include karni padti hai. Template syntax ke through kisi bhi data type ka vector banaya ja sakta hai.

Operation Syntax Time Complexity Description
Declaration vector<int> arr; O(1) Empty vector create karta hai
push_back arr.push_back(x); O(1) amortized End mein element insert karta hai
pop_back arr.pop_back(); O(1) Last element remove karta hai
size arr.size(); O(1) Current number of elements return karta hai
front arr.front(); O(1) First element return karta hai
back arr.back(); O(1) Last element return karta hai
empty arr.empty(); O(1) True return karta hai agar vector empty hai
Random Access arr[i]; O(1) i-th index pe direct access

Code Implementation

// Vector Fundamentals Demo #include <iostream> #include <vector> // Vector use karne ke liye ye header file include karni padti hai using namespace std; int main() { // Vector Declaration vector<int> arr; // Vector me Elements Insert karna // push_back() hamesha last me element insert karta hai arr.push_back(10); // arr = [10] arr.push_back(20); // arr = [10, 20] arr.push_back(30); // arr = [10, 20, 30] arr.push_back(40); // arr = [10, 20, 30, 40] // Last Element Remove karna // pop_back() sirf last element ko remove karta hai arr.pop_back(); // arr = [10, 20, 30] // Vector ka Size check karna cout << "Size : " << arr.size() << endl; // Vector ke Elements Print karna cout << "Elements : "; for (int i = 0; i < arr.size(); i++) { cout << arr[i] << " "; } cout << endl; // First aur Last Element Access karna cout << "First Element : " << arr.front() << endl; cout << "Last Element : " << arr.back() << endl; // Check karo Vector Empty hai ya nahi if (arr.empty()) { cout << "Vector is Empty" << endl; } else { cout << "Vector is Not Empty" << endl; } return 0; }

Output

Size : 3 Elements : 10 20 30 First Element : 10 Last Element : 30 Vector is Not Empty

Common Mistakes

Edge Case
  • Empty vector pe front() / back(): Empty vector pe front() ya back() call karna undefined behavior hai. Hamesha empty() check karo.
  • pop_back() on empty vector: Ye bhi undefined behavior hai. Pehle size check karo.
  • Out of bounds access: arr[i] se bounds check nahi hota. arr.at(i) use karo agar exception chahiye.
  • push_back vs emplace_back: emplace_back objects ke liye efficient hai, lekin primitives ke liye dono same hain.
Interview Tip Vector ke internal working ko samajhna zaroori hai: jab capacity full hoti hai, vector double capacity allocate karta hai. Isliye push_back ka amortized time O(1) hota hai. Agar exact size pata hai, toh vector<int> arr(n); se initialize karo taaki reallocation avoid ho.
Key Takeaways
  • Vector dynamic array hai jo STL ka part hai.
  • push_back end mein insert karta hai, pop_back last element remove karta hai.
  • Random access O(1) time mein possible hai.
  • Empty vector pe front(), back(), pop_back() se bacho.

Search an Element in an Array

Pattern Searching — Linear Search, Array Traversal, Brute Force

Introduction

Ye problem mein ek array aur ek target number diya jata hai. Humein target ka index return karna hai agar wo array mein exist karta hai, nahi toh -1 return karna hai. Since array sorted nahi hai, hum Linear Search use karenge — har element ko ek-ek karke check karenge.

The Analogy

Jaise aapko ek room mein kisi specific person ko dhoondhna hai aur aapko pata nahi ki wo kahan baitha hai. Aap har ek person se poochhte jaoge “kya aap target ho?” jab tak wo person nahi mil jata. Ye exactly Linear Search ka concept hai.

How It Works Internally

  1. Array ke starting index se loop chalao.
  2. Har index pe current element ko target se compare karo.
  3. Agar match mil jaye, toh current index return karo.
  4. Loop complete hone par target nahi mila, toh -1 return karo.
Workflow Step 1: i = 0 se n-1 tak iterate karo.
Step 2: Check karo arr[i] == target.
Step 3: Match mila? Return i.
Step 4: Loop end? Return -1.

Code Implementation

// GFG Problem: Search an Element in an Array class Solution { public: int search(int arr[], int n, int target) { // Array ke saare elements ko traverse karo for (int i = 0; i < n; i++) { // Agar target mil gaya to uska index return kar do if (arr[i] == target) { return i; } } // Agar target poore array me nahi mila return -1; } };

Complexity Analysis

Case Time Complexity Reason
Best O(1) Target first element pe hi mil jaye.
Average O(n) Target kisi bhi random position pe ho sakta hai.
Worst O(n) Target last element pe hai ya array mein nahi hai.
Space O(1) Sirf loop variable use hota hai.
Interview Tip Agar array sorted hai, toh Linear Search mat lagao. Us case mein Binary Search use karo jo O(log n) time mein kaam karta hai. Interviewer hamesha optimized approach expect karta hai jab sorted data ki baat ho.
Key Takeaways
  • Linear Search unsorted arrays ke liye suitable hai.
  • Time complexity O(n), space O(1).
  • First occurrence return hoti hai agar duplicates hain.
  • Sorted arrays ke liye Binary Search preferable hai.

Reverse an Array

Pattern Two Pointers — Array Manipulation, In-Place Swap, Symmetric Traversal

Introduction

Array reverse karne ke liye hum Two Pointer Technique use karte hain. First element ko last element se, second element ko second last se, aur aise hi opposite elements ko swap karte jate hain. Ye approach in-place hai aur O(1) extra space use karti hai.

The Analogy

Socho aapke paas ek row mein books rakhi hain. Aap pehli book ko last book ki jagah rakhte ho, second ko second last ki jagah, aur aise hi beech mein milte ho. Jab aap center tak pahunchte ho, poori row ulat chuki hoti hai. Exactly aise hi Two Pointer approach kaam karta hai.

How It Works Internally

  1. Do pointers initialize karo: i = 0 (start) aur j = n - 1 (end).
  2. Tab tak swap karo jab tak i < j.
  3. Har swap ke baad i++ aur j-- karo.
  4. Center tak pahunchne par array reverse ho chuki hoti hai.
Workflow Initial: [1, 2, 3, 4, 5]   i=0, j=4
Swap 1: [5, 2, 3, 4, 1]   i=1, j=3
Swap 2: [5, 4, 3, 2, 1]   i=2, j=2
Stop: i >= j, loop ends.

Why Only n/2 Iterations?

Ek swap mein do elements apni correct position pe pahunch jate hain. Agar hum n/2 ke baad bhi swap karenge, toh wahi elements dobara swap ho jayenge aur array wapas original order mein aa jayegi. Isliye sirf n/2 tak hi loop chalana chahiye.

Code Implementation

// GFG Problem: Reverse an Array class Solution { public: void reverseArray(int arr[], int n) { // Sirf n/2 tak hi loop chalega // Kyunki ek swap me 2 elements apni correct position par aa jate hai for (int i = 0; i < n / 2; i++) { // Current element ko uske opposite element se swap karo int temp = arr[i]; arr[i] = arr[n - 1 - i]; arr[n - 1 - i] = temp; } } };

Complexity Analysis

Case Time Complexity Reason
All Cases O(n) n/2 swaps required hain.
Space O(1) Sirf temp variable use hota hai, in-place reversal.
Best Practice Two Pointer technique ko rote learn mat karo, logic samjho. Ye technique palindrome check, two sum, sorted array pair sum jaise problems mein bhi use hoti hai. Pattern recognize karna seekho.
Key Takeaways
  • Two Pointer technique in-place reversal karti hai.
  • Sirf n/2 iterations chahiye kyunki har swap mein do elements fix hote hain.
  • Time O(n), Space O(1).
  • Ye technique dusre problems mein bhi apply hoti hai.

Check if Array is Sorted

Pattern Validation — Linear Traversal, Adjacent Comparison, Early Exit

Introduction

Is problem mein humein check karna hai ki kya given array ascending order mein sorted hai ya nahi. Iske liye hum adjacent elements ko compare karenge. Agar kisi bhi point pe current element previous element se chhota mil jaye, toh array sorted nahi hai.

How It Works Internally

  1. Index 1 se last tak iterate karo.
  2. Har step mein arr[i] ko arr[i-1] se compare karo.
  3. Agar arr[i] < arr[i-1], toh false return karo.
  4. Loop complete hone par true return karo.
Workflow Check: arr[i] >= arr[i-1] for all i from 1 to n-1.
Violation: Kahi bhi arr[i] < arr[i-1] mila? Return false.
Success: Poora loop complete? Return true.

Code Implementation

// GFG Problem: Check if Array is Sorted class Solution { public: bool arraySortedOrNot(int arr[], int n) { // Index 1 se start karenge kyunki pehle element ka // koi previous element nahi hota for (int i = 1; i < n; i++) { // Current element ko previous element se compare karo if (arr[i] < arr[i - 1]) { return false; } } // Kahin bhi order break nahi hua return true; } };

Complexity Analysis

Case Time Complexity Reason
Best O(1) Second element pe hi violation mil jaye.
Worst O(n) Array sorted hai ya last element pe violation hai.
Space O(1) Sirf loop variable.
Key Takeaways
  • Adjacent comparison se sorted check hota hai.
  • Early exit possible hai — violation milte hi return karo.
  • Time O(n) worst case, O(1) best case.
  • Descending order check karne ke liye arr[i] > arr[i-1] condition badlo.

Missing Number in Array

Pattern Array Mathematics — Brute Force, Sum Formula, Bit Manipulation (XOR)

Introduction

Array mein 0 se n tak ke numbers hain lekin ek number missing hai. Humein wo missing number dhoondhna hai. Is problem ko solve karne ke teen approaches hain: Brute Force, Sum Formula, aur XOR. Har approach ki complexity different hai aur ye interview mein commonly pucha jata hai.

Key Insight: Missing Number problem ko multiple tareeqon se solve kiya ja sakta hai. Sum Formula most optimal hai O(n) time, O(1) space mein. XOR approach bhi same complexity deta hai lekin overflow se bachata hai.

Approach 1: Brute Force

Basic

Har expected number 0 se n tak ko array mein search karo. Jo number nahi mile, wahi missing hai.

Workflow Outer Loop: i = 0 to n (expected numbers).
Inner Loop: j = 0 to n-1 (array search).
Check: arr[j] == i?
Found: Break inner loop. Next i.
Not Found: Return i.
// Approach 1: Brute Force (Nested Loop) class Solution { public: int missingNumber(vector<int>& arr) { int n = arr.size(); // 0 se n tak saare expected numbers check karo for (int i = 0; i <= n; i++) { bool found = false; // Current expected number ko poore array me search karo for (int j = 0; j < n; j++) { if (arr[j] == i) { found = true; break; } } // Agar number nahi mila to wahi missing number hai if (!found) { return i; } } return -1; } };
TimeSpaceDrawback
O(n²)O(1)Nested loop ki wajah se slow hai.

Approach 2: Sum Formula (Optimal)

Intermediate

Mathematical formula use karo: Sum of 0 to n = n*(n+1)/2. Is expected sum mein se actual array ka sum minus kar do. Difference hi missing number hoga.

Workflow Step 1: totalSum = n * (n + 1) / 2.
Step 2: actualSum = array ke saare elements ka sum.
Step 3: missingNumber = totalSum - actualSum.
Return: missingNumber.
// Approach 2: Sum Formula (Optimal) class Solution { public: int missingNumber(vector<int>& arr) { int n = arr.size(); int actualSum = 0; int totalSum = 0; // Calculate Actual Sum for (int i = 0; i < n; i++) { actualSum += arr[i]; } // Calculate Expected Sum (0 to n) totalSum = n * (n + 1) / 2; // Missing Number = Expected Sum - Actual Sum return totalSum - actualSum; } };
TimeSpaceAdvantage
O(n)O(1)Single pass mein result, very efficient.
Important Integer Overflow Risk: Agar n bahut bada hai, toh n*(n+1)/2 integer overflow kar sakta hai. Is case mein long long use karo ya XOR approach prefer karo.

Approach 3: XOR (Bit Manipulation)

Advanced

XOR ki property use karte hain: a ^ a = 0 aur a ^ 0 = a. Expected numbers (0 to n) aur actual array numbers dono ka XOR karne par same numbers cancel ho jayenge. Jo number sirf ek baar bachega, wahi missing number hoga.

Workflow Property: a ^ a = 0, a ^ 0 = a.
Step 1: ans = n (last expected number).
Step 2: Loop i = 0 to n-1: ans ^= i ^ arr[i].
Result: ans mein missing number bachega.
// Approach 3: XOR (Bit Manipulation) class Solution { public: int missingNumber(vector<int>& arr) { int n = arr.size(); // ans ko n se initialize kar diya, // taki expected numbers me last number (n) cover ho jaye int ans = n; // Ek hi loop me Expected Numbers aur Actual Numbers dono ka XOR for (int i = 0; i < n; i++) { ans ^= i; // Expected Number ka XOR ans ^= arr[i]; // Actual Number ka XOR } // Saare common numbers cancel ho jayenge, // sirf missing number bachega return ans; } };
TimeSpaceAdvantage
O(n)O(1)Overflow safe, bit manipulation skill showcase.

Comparative Analysis

Approach Time Space When to Use
Brute Force O(n²) O(1) Never in interviews. Only for understanding.
Sum Formula O(n) O(1) Quick implementation, small n ke liye best.
XOR O(n) O(1) Large n, overflow concern, bit manipulation demo.
Interview Tip Missing Number problem interview mein bahut common hai. Hamesha Sum Formula approach pehle batana, phir XOR approach batana. Ye dikhata hai ki aap multiple solutions sochte ho aur trade-offs samajhte ho. XOR approach bit manipulation ka knowledge bhi demonstrate karta hai.
Key Takeaways
  • Missing Number ke 3 approaches hain: Brute Force, Sum Formula, XOR.
  • Sum Formula O(n) time, O(1) space mein kaam karta hai.
  • XOR approach overflow safe hai aur bit manipulation demonstrate karta hai.
  • Brute Force nested loop se O(n²) hai, avoid karo.
