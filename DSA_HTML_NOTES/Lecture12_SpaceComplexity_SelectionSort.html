Part 1: Space Complexity Deep Dive

What is Space Complexity?

Space Complexity = Input Space + Auxiliary Space. Lekin generally jab hum Space Complexity bolte hain, toh Auxiliary Space ki baat karte hain — matlab algorithm ke andar jo extra space use ho rahi hai.

Workflow / Algorithm

Space Complexity Calculate Karne Ka Tarika:

  1. Code mein dekho ki kitne extra variables declare ho rahe hain.
  2. Agar new array bana rahe ho toh uski size count karo.
  3. Recursion mein call stack ki memory bhi add karo.
  4. Input array ki memory ignore karo — woh toh already given hai.
  5. Sabse bade extra space wale term ko pakad lo — wahi S.C. hoti hai.

Example 3: In-Place Reverse (O(1) Space)

// In-place reverse — no extra array needed int i = 0, j = n - 1; while (i < j) { swap(arr[i], arr[j]); i++; j--; }
Best Practice / Optimization

Yeh approach Two Pointer Technique use karti hai. Sirf do variables i aur j extra use ho rahe hain — chahe array kitna bada ho. Isliye S.C. = O(1). Hamesha try karo ki in-place solve karo, extra array mat banao.

// Recursive function — each call uses stack space int sum(int n) { if (n == 0) return 0; return n + sum(n - 1); }

Part 2: Selection Sort Algorithm

Real-World Analogy

Socho class mein 5 students line mein khade hain unki height ke hisaab se: [10, 3, 9, 9, 5]. Teacher ko sabse chhote student ko sabse aage laana hai. Phir bache hue mein se phir sabse chhote ko second position pe, and so on. Yahi logic Selection Sort ka hai — har step pe minimum element dhoondh ke usko correct position pe lao.

Problem Statement

Given array [10, 3, 9, 9, 5] ko ascending order mein sort karna hai using Selection Sort. Step-by-step dry run karo, code likho, aur complexity analyze karo.

Algorithm Steps

Workflow / Algorithm
  1. Outer loop i = 0 se n-2 tak chalega.
  2. Har i ke liye, assume karo ki i wala index minimum hai.
  3. Inner loop j = i+1 se n-1 tak — remaining array mein minimum dhoondo.
  4. Agar nums[j] < nums[minIndex], toh minIndex = j.
  5. Inner loop ke baad, swap(nums[i], nums[minIndex]).
  6. Repeat until sorted.

Complete C++ Implementation

class Solution { public: vector<int> sortArray(vector<int>& nums) { int n = nums.size(); // Selection Sort for (int i = 0; i < n - 1; i++) { // Assume current index has the smallest element int minIndex = i; for (int j = i + 1; j < n; j++) { if (nums[j] < nums[minIndex]) { minIndex = j; } } // Swap current element with minimum found swap(nums[i], nums[minIndex]); } return nums; } };

Time & Space Complexity Analysis

Complexity Analysis

Time Complexity:

Outer loop: n-1 iterations
Inner loop: (n-1) + (n-2) + ... + 1
Sum = n(n-1)/2 = (n² - n)/2
T.C. = O(n²)

Space Complexity:

Variables: i, j, minIndex
No extra array used
In-place sorting
S.C. = O(1)
Key Insight: Selection Sort mein hamesha O(n²) comparisons hote hain chahe array sorted ho ya ulta. Kyunki har pass mein inner loop poora chalta hai minimum dhoondhne ke liye. Isliye Best Case = Worst Case = Average Case = O(n²). Lekin swaps sirf O(n) hote hain — har pass mein maximum 1 swap.

Big-O, Omega, Theta Revisited

O Big-O (Upper Bound) — Sabse Zyada

Big-O batata hai ki algorithm maximum kitna time lega. Agar T.C. = n² hai, toh O(n²), O(n³), O(n⁴) sab valid upper bounds hain. Lekin hum tightest upper bound dete hain — matlab O(n²).

Interview Rule: Hamesha Big-O batana. O(n²) ke jagah O(n³) mat likhna — interviewer ko pata chal jaata hai ki tumhe exact complexity nahi aati.

Ω Omega (Lower Bound) — Sabse Kam

Omega batata hai ki algorithm minimum kitna time lega. Agar T.C. = n² hai, toh Ω(n²), Ω(n), Ω(1) sab valid lower bounds hain. Lekin tightest lower bound Ω(n²) hota hai.

Θ Theta (Tight Bound) — Exact

Theta tab use hota hai jab upper bound = lower bound. Selection Sort ke case mein T.C. hamesha n² hai — chahe best case ho ya worst. Isliye Θ(n²) bhi valid hai. Lekin practically sirf Big-O use karte hain.

Important

Selection Sort mein teeno cases same hain — Best, Average, Worst sab O(n²). Kyunki chahe array sorted ho ya nahi, inner loop hamesha poora chalta hai minimum dhoondhne ke liye. Yeh Bubble Sort se behtar isliye hai kyunki swaps kam hote hain (maximum n swaps), lekin comparisons same hain.

Interview Tips & Common Pitfalls

Interview Tip
  • Selection Sort ka dry run zaroor practice karo — interview mein pen-paper pe dry run mangte hain.
  • Space Complexity puchein toh auxiliary space batana — input array ki memory nahi ginte.
  • Selection Sort stable nahi hai — equal elements ka original order change ho sakta hai.
  • Swaps vs Comparisons alag hote hain — Selection Sort mein comparisons O(n²) lekin swaps O(n).
  • Big-O mein hamesha tightest bound do — O(n²) ke jagah O(n³) mat likhna.
Edge Case
  • n = 0 ya n = 1 — already sorted, loop nahi chalega.
  • Duplicate values — Selection Sort duplicates ko handle kar leta hai lekin stable nahi hai.
  • Already sorted array — phir bhi O(n²) comparisons honge (yeh disadvantage hai).
  • Large arrays ke liye Selection Sort avoid karo — Merge Sort ya Quick Sort use karo.

Key Takeaways

Takeaway
  1. Space Complexity = Extra space jo algorithm use karta hai. Input space nahi ginte.
  2. Simple variables → O(1); New array → O(n); Recursion → O(n) stack space.
  3. In-place algorithms (Two Pointer) hamesha behtar hote hain — extra memory bachati hai.
  4. Selection Sort ka logic: Har pass mein minimum dhoondh ke usko correct position pe lao.
  5. Selection Sort: T.C. = O(n²), S.C. = O(1) — all cases same.
  6. Swaps maximum n-1 hote hain — isliye write-heavy systems mein useful ho sakta hai.
  7. Sorting real life mein har jagah — e-commerce, maps, databases, spreadsheets.
  8. Big-O = Upper Bound (sabse zyada), Omega = Lower Bound (sabse kam), Theta = Tight Bound (exact).
