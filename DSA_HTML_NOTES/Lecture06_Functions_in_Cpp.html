Lecture 06: Functions in C++

Is lecture mein hum Functions ko ground zero se samajhenge — kya hai, kaise kaam karta hai, aur kyun zaroori hai. Functions programming ka woh block hai jo reusable code likhne ki power deta hai. Agar aap first time functions padh rahe ho, toh ye notes aapke liye perfect starting point hai.

❦ ❦ ❦

Why Functions? — The DRY Principle

Key Insight: DRY ka matlab hai — "Do Not Repeat Yourself". Agar aapko same code baar-baar likhna pad raha hai, toh usse ek function bana lo aur baar-baar use karo.

The Problem Without Functions

Socho aapko 30 alag-alag numbers ka factorial nikalna hai. Bina functions ke aapko same factorial code 30 baar copy-paste karna padega. Agar code mein koi bug hai, toh 30 jagah fix karna padega. Ye inefficient hai aur galatiyan hone ka chance bahut zyada hai.

Important
  • Without Functions: Same code 30 baar likhna → 30 bugs fix karna → Time waste
  • With Functions: Code ek baar likho → 30 baar call karo → Bug ek jagah fix karo

Real-World Analogy

Function ko ek machine ki tarah samjho. Jaise washing machine — aap kapde daalo, button dabao, aur machine apna kaam karti hai. Aapko baar-baar washing machine ka internal mechanism nahi samajhna padta. Bas input do, aur output lo. Function bhi waise hi kaam karta hai.

❦ ❦ ❦

What is a Function?

Function ek named block of code hai jo ek specific task perform karta hai. Jab function ko call karte hain, toh wo apna kaam karke result de sakta hai.

Workflow
  • Function Definition: Function ka code likhna (kya kaam karega)
  • Function Call: Function ko use karna (jab jarurat ho)
  • Function Return: Result wapas dena (agar kuch calculate kiya ho)

Function Syntax — Step by Step

// Step 1: Function Definition return_type function_name(parameters) { // Function body — jo kaam karna hai return value; // (agar kuch return karna ho) } // Step 2: Function Call function_name(arguments);
Intuition
  • Return Type: Function kya type ki value wapas dega? (int, double, void — agar kuch nahi dega)
  • Function Name: Function ko identify karne ka naam
  • Parameters: Function ko input dene ke liye variables (jaise machine mein kapde daalna)
  • Arguments: Actual values jo function call ke time pass hoti hain
❦ ❦ ❦

Example 1: Factorial Function (void — No Return)

Pehle dekhte hain ek simple function jo sirf print karta hai, kuch return nahi karta. Isme void return type use hota hai.

#include <iostream> using namespace std; // Function Definition void fact(int n) { // void = kuch return nahi karega int factorial = 1; for(int i = 1; i <= n; i++) { factorial *= i; } cout << "Factorial of " << n << " is: " << factorial << endl; } int main() { fact(5); // Function Call with Argument 5 fact(7); // Function Call with Argument 7 fact(3); // Function Call with Argument 3 return 0; }
Workflow
StepWhat Happens
1main() start hota hai
2fact(5) call hota hai — control fact function mein jaata hai
3n = 5 set hota hai, factorial calculate hota hai = 120
4Result print hota hai, control main() mein wapas aata hai
5fact(7) call hota hai — same process repeat
6fact(3) call hota hai — same process repeat

Example 2: Factorial Function (int — With Return)

Ab dekhte hain ek better version jo factorial calculate karke wapas return karta hai. Isse hum calculated value ko store kar sakte hain, compare kar sakte hain, ya further calculation mein use kar sakte hain.

#include <iostream> using namespace std; // Function with return type int int factorial(int n) { int fact = 1; for(int i = 1; i <= n; i++) { fact *= i; } return fact; // Calculated value wapas bhej raha hai } int main() { int result = factorial(5); // Return value ko store kiya cout << "Factorial: " << result << endl; // Ab is result ko kahin bhi use kar sakte ho if(result > 100) { cout << "Greater than 100" << endl; } return 0; }
Key Insight: Return value ko main function mein store karke aap uspe further operations kar sakte ho — comparison, arithmetic, ya kisi aur function mein pass karna. void function mein ye flexibility nahi hoti.
❦ ❦ ❦

Example 3: Grade Printing Function

Ek practical example jisme function marks ke basis pe grade print karta hai. Ye dekho kaise multiple conditions ek function mein handle hoti hain.

#include <iostream> using namespace std; void findGrade(int marks) { if(marks >= 90) { cout << "Grade: A+" << endl; } else if(marks >= 80) { cout << "Grade: A" << endl; } else if(marks >= 70) { cout << "Grade: B" << endl; } else if(marks >= 60) { cout << "Grade: C" << endl; } else { cout << "Grade: F (Fail)" << endl; } } int main() { findGrade(95); // A+ findGrade(82); // A findGrade(45); // F return 0; }
❦ ❦ ❦

Multiple Parameters

Function mein ek se zyada parameters bhi pass kar sakte ho. Har parameter comma se separate hota hai.

#include <iostream> using namespace std; // Function with multiple parameters void sum(int a, int b) { cout << "Sum: " << a + b << endl; } int main() { sum(10, 20); // a=10, b=20 → Sum: 30 sum(5, 7); // a=5, b=7 → Sum: 12 return 0; }
Intuition Jab sum(10, 20) call hota hai, toh pehla argument pehle parameter mein jaata hai, aur dusra argument dusre parameter mein. Order matter karta hai — sum(10, 20) aur sum(20, 10) same nahi hote.
❦ ❦ ❦

Example 4: Square Function with Return

Ek simple function jo number ka square return karta hai. Dekho kaise return value ko directly cout mein bhi use kar sakte hain.

#include <iostream> using namespace std; // Function definition int square(int n) { return n * n; // Square calculate karke wapas bhej raha hai } int main() { int num = 5; // Method 1: Return value ko store karke print karo int result = square(num); cout << "Square: " << result << endl; // Method 2: Directly cout mein function call karo cout << "Square of 7: " << square(7) << endl; return 0; }

Pass by Value vs Pass by Reference

Ye C++ ka sabse important concept hai. Jab function ko value pass karte hain, toh do tarike hote hain — Pass by Value aur Pass by Reference. Inka difference samajhna interviews mein bahut zaroori hai.

Pass by Value (Default)

Intuition Pass by Value mein function ko original variable ki copy milti hai. Function us copy pe kaam karta hai, original variable unchanged rehta hai. Jaise aap kisi ko apni photo ki copy doge, wo uspe kuch bhi likhe — aapki original photo safe rahegi.
#include <iostream> using namespace std; void increment(int a) { // 'a' is a COPY of the original variable a++; // Only the copy is incremented cout << "Inside function: a = " << a << endl; } int main() { int num = 10; cout << "Before: num = " << num << endl; increment(num); // num ki COPY ja rahi hai cout << "After: num = " << num << endl; return 0; }

Memory Layout — Pass by Value

🔷 MEMORY LAYOUT: Pass by Value

main() Function increment() Function
Address 1000 num = 10 → Address 2000 a = 10 (COPY)
Address 1000 num = 10 → Address 2000 a = 11 (COPY changed)
Address 1000 num = 10 ✓ ← Address 2000 a = 11 (destroyed)

✅ Original num unchanged — sirf COPY modify hui

Workflow — Behind the Scenes
LocationVariableValue
main()num10
increment()a (copy of num)10 → 11
main() (after call)num10 (unchanged!)
Important Pass by Value mein original variable kabhi change nahi hota. Function sirf copy pe kaam karta hai. Ye safe hai lekin agar aapko original value change karni hai, toh ye kaam nahi karega.

Pass by Reference

Intuition Pass by Reference mein function ko original variable ka address milta hai (na ki copy). & symbol use hota hai parameter ke aage. Function jo bhi change karega, wo original variable mein reflect hoga. Jaise aap kisi ko apna original photo doge — wo uspe kuch bhi likhega toh aapki photo permanently change ho jayegi.
#include <iostream> using namespace std; void increment(int &a) { // '&' means reference — original variable ka address a++; // Original variable incremented! cout << "Inside function: a = " << a << endl; } int main() { int num = 10; cout << "Before: num = " << num << endl; increment(num); // num ka ADDRESS ja raha hai cout << "After: num = " << num << endl; return 0; }

Memory Layout — Pass by Reference

🔷 MEMORY LAYOUT: Pass by Reference

main() Function increment() Function

Address 1000 num = 11 ✓ ↔ &a → 1000 a++ → num becomes 11 (SAME memory)
Address 1000 num = 11 ✓ ← &a → 1000 reference ended

⚠️ Original num changed — same memory location modify hui

Workflow — Behind the Scenes

Key Insight: Pass by Value mein copy banti hai — original safe rehta hai. Pass by Reference mein address jaata hai — original change ho jata hai. & symbol yaad rakho — "& means ANDar ki value change hogi".

Example 5: Swap Two Numbers (Pass by Reference)

Swap function Pass by Reference ka best example hai. Bina reference ke swap kaam nahi karega kyunki copy pe swap karne se original unchanged rehta hai.

#include <iostream> using namespace std; // Pass by Reference — original values swap hongi void swap(int &a, int &b) { int temp = a; a = b; b = temp; } int main() { int x = 10, y = 20; cout << "Before: x = " << x << ", y = " << y << endl; swap(x, y); // x aur y swap ho jayenge cout << "After: x = " << x << ", y = " << y << endl; return 0; }

Memory Layout — Swap Function

🔷 MEMORY LAYOUT: Swap with Pass by Reference

Step x (0x1000) y (0x1004) temp
Initial 10 20 —
temp = a 10 20 10
a = b 20 20 10
b = temp 20 10 10

✅ x = 20, y = 10 — Values successfully swapped!

Edge Case Agar aap swap function ko Pass by Value se likhoge (void swap(int a, int b)), toh x aur y swap nahi honge. Sirf copies swap hongi, originals same rahenge. Ye common interview mistake hai!

Function Overloading

Function Overloading ka matlab hai same naam ke multiple functions banana, lekin unke parameters alag hone chahiye. C++ compiler parameters ke basis pe decide karta hai ki kaunsa function call karna hai.

Intuition Socho aapke ghar mein sabka naam "Kumar" hai — lekin ek Kumar doctor hai, ek Kumar engineer hai, ek Kumar teacher hai. Same naam, alag kaam. Function overloading bhi waise hi kaam karta hai — same function name, alag parameters.
#include <iostream> using namespace std; // Function 1: Two integers ka sum int sum(int a, int b) { return a + b; } // Function 2: Three integers ka sum (different number of parameters) int sum(int a, int b, int c) { return a + b + c; } // Function 3: Two floats ka sum (different parameter types) float sum(float a, float b) { return a + b; } int main() { cout << sum(3, 4) << endl; // Calls Function 1 → 7 cout << sum(3, 4, 5) << endl; // Calls Function 2 → 12 cout << sum(3.5f, 4.5f) << endl; // Calls Function 3 → 8.0 return 0; }
Workflow — How Compiler Decides
CallArgumentsMatched FunctionReason
sum(3, 4)int, intsum(int, int)Exact match
sum(3, 4, 5)int, int, intsum(int, int, int)3 parameters
sum(3.5f, 4.5f)float, floatsum(float, float)float types
Important Function Overloading mein sirf return type alag karne se kaam nahi chalega. Parameters ka number ya type alag hona zaroori hai. int sum(int, int) aur float sum(int, int) — ye overloading nahi hai, ye compile error dega.
❦ ❦ ❦

Default Parameters

Default Parameters se aap function call ke time kuch arguments skip kar sakte ho. Agar argument nahi diya, toh default value use hoti hai.

#include <iostream> using namespace std; // Default parameter: num = 5 void printing(int num = 5) { cout << "Number: " << num << endl; } int main() { printing(10); // Argument diya → num = 10 printing(); // Argument nahi diya → num = 5 (default) return 0; }
Workflow
CallArgument Passed?Value of num
printing(10)Yes10 (argument used)
printing()No5 (default used)
Best Practice Default parameters hamesha rightmost se start karo. void fn(int a, int b = 5, int c = 10) ✓ sahi hai. void fn(int a = 5, int b) ✗ galat hai — default parameter ke baad non-default nahi aa sakta.

Real-World Use Case: YouTube Video System

Ek real-world example samajhte hain — YouTube video upload system. Har video ke liye same steps hote hain: thumbnail generate karna, title set karna, views count karna, upload time record karna. Ye sab ek function mein daal do aur har video ke liye call karo.

#include <iostream> using namespace std; // Function jo har video ke liye same kaam karega void uploadVideo(string title, string thumbnail, int views) { cout << "========================================" << endl; cout << "Thumbnail: " << thumbnail << endl; cout << "Title: " << title << endl; cout << "Views: " << views << endl; cout << "Upload Time: " << "Just now" << endl; cout << "========================================" << endl; } int main() { // 4 videos upload kar rahe hain uploadVideo("C++ Basics", "thumb1.jpg", 1500); uploadVideo("Arrays in C++", "thumb2.jpg", 2300); uploadVideo("Pointers Explained", "thumb3.jpg", 8900); uploadVideo("OOP Concepts", "thumb4.jpg", 5600); return 0; }
Intuition Agar uploadVideo function nahi hota, toh 4 videos ke liye same code 4 baar likhna padta. Function ne code ko reusable bana diya. Kal ko agar upload logic change hua (jaise "Upload Time" ke format mein), toh sirf ek jagah change karna padega.

Key Takeaways

Takeaway
  • DRY Principle: "Do Not Repeat Yourself" — same code baar-baar mat likho, function bana lo.
  • Function = Machine: Input do, processing hogi, output lo. Internal mechanism jaanna zaroori nahi.
  • Return Type: void = kuch return nahi karta. int, double, etc. = value return karta hai.
  • Parameters vs Arguments: Parameters = function definition mein variables. Arguments = function call ke time actual values.
  • Pass by Value: Copy jaati hai, original safe rehta hai. Default behavior hai.
  • Pass by Reference: Address jaata hai, original change ho jata hai. & symbol use karo.
  • Swap Function: Hamesha Pass by Reference se likho, warna swap nahi hoga.
  • Function Overloading: Same naam, alag parameters. Return type alag karne se overloading nahi hoti.
  • Default Parameters: Rightmost se start karo. Call ke time skip kar sakte ho.
