Problem 1 — GFG: Cyclically Rotate an Array by One

Problem Statement

Array ke sabhi elements ko ek position right shift karna hai aur last element ko first position par lana hai.

Example: [1,2,3,4,5] → [5,1,2,3,4]

Algorithm

1. Last element ko save kar lo.
2. Right se left tak sabhi elements ko ek position shift karo.
3. Saved element ko first index par rakh do.

void rotate(vector<int> &arr) { int n = arr.size(); // Step 1: Last element save kar lo int last = arr[n - 1]; // Step 2: Right shift (peeche se aage ki taraf) for (int i = n - 1; i > 0; i--) { arr[i] = arr[i - 1]; } // Step 3: Saved element first par rakho arr[0] = last; }

Problem 2 — LeetCode 189: Rotate Array by K

Problem Statement

Given an integer array nums, rotate the array to the right by k steps, where k is non-negative.

Example: nums = [1,2,3,4,5,6,7], k = 3 → [5,6,7,1,2,3,4]

Critical Insight

Agar array size n hai aur k >= n ho, to effective rotations = k % n hote hain. Kyunki n rotations karne par array wapas original position me aa jata hai.

Approach 1: Brute Force O(n×k)

Algorithm

Rotate by One ko k baar repeat karo. Har rotation me last element save karke right shift karte hain.

void rotate(vector<int>& nums, int k) { int n = nums.size(); k = k % n; // Effective rotations // Rotate by One, k Times for (int i = 0; i < k; i++) { int last = nums[n - 1]; for (int j = n - 1; j > 0; j--) { nums[j] = nums[j - 1]; } nums[0] = last; } }

Approach 2: Temporary Array O(n)

Intuition / Insight

Har element ki new position formula se nikal sakte hain: newIndex = (currentIndex + k) % n. Ek temporary array me correct position pe place karo, phir copy back.

void rotate(vector<int>& nums, int k) { int n = nums.size(); k = k % n; vector<int> temp(n); // Har element ko uski new position par rakho for (int i = 0; i < n; i++) { temp[(i + k) % n] = nums[i]; } // Copy back into original array for (int i = 0; i < n; i++) { nums[i] = temp[i]; } }

Approach 3: Reverse Algorithm (Optimal) O(n) — Best

Intuition / Insight

3 reversals se kaam ho jata hai:
1. Poora array reverse karo.
2. First k elements reverse karo.
3. Remaining (n-k) elements reverse karo.

Ye trick mathematically prove ki ja sakti hai — har element apni correct position pe aa jata hai.

// Using STL reverse() void rotate(vector<int>& nums, int k) { int n = nums.size(); k = k % n; // Step 1: Reverse complete array reverse(nums.begin(), nums.end()); // Step 2: Reverse first k elements reverse(nums.begin(), nums.begin() + k); // Step 3: Reverse remaining elements reverse(nums.begin() + k, nums.end()); }
// Without STL — Custom Reverse Function void reverseArray(vector<int>& nums, int left, int right) { while (left < right) { swap(nums[left], nums[right]); left++; right--; } } void rotate(vector<int>& nums, int k) { int n = nums.size(); k = k % n; reverseArray(nums, 0, n - 1); // Full reverse reverseArray(nums, 0, k - 1); // First k reverse reverseArray(nums, k, n - 1); // Remaining reverse }

Problem 3 — LeetCode 1748: Sum of Unique Elements

Problem Statement

Array me jo elements sirf ek baar aate hain, un sabka sum return karna hai.

Example 1: [1,2,3,2] → Unique = 1,3 → Sum = 4
Example 2: [1,1,1,1,1] → No unique → Sum = 0

Approach 1: Brute Force O(n²)

int sumOfUnique(vector<int>& nums) { int n = nums.size(); int sum = 0; // Har element ki frequency count karo for (int i = 0; i < n; i++) { int freq = 0; for (int j = 0; j < n; j++) { if (nums[i] == nums[j]) freq++; } // Sirf unique element add karo if (freq == 1) sum += nums[i]; } return sum; }

Approach 2: Frequency Array (Optimal) O(n)

Intuition / Insight

Constraints ke hisaab se (1 <= nums[i] <= 100) ek fixed-size frequency array banake har element ki frequency store kar lo. Phir dobara traverse karke jiski frequency 1 ho usse add kar do.

int sumOfUnique(vector<int>& nums) { // Frequency Array (constraint: 1 to 100) vector<int> freq(101, 0); // Step 1: Frequency Count for (int i = 0; i < nums.size(); i++) { freq[nums[i]]++; } int sum = 0; // Step 2: Unique Elements ka Sum for (int i = 0; i < nums.size(); i++) { if (freq[nums[i]] == 1) { sum += nums[i]; } } return sum; }
Best Practice / Optimization

Agar constraints bounded ho (jaise 1-100), to frequency array (counting sort technique) use karo — O(n) time, O(1) space (fixed 101 size). Agar range bada ho to unordered_map use karo.

Complexity Analysis
ApproachTimeSpace
Brute ForceO(n²)O(1)
Frequency ArrayO(n)O(1)

Problem 4 — GFG: Second Largest Element

Problem Statement

Array me Largest Element ke baad jo sabse bada distinct element hota hai, use Second Largest kehte hain. Agar exist na kare to -1 return karo.

Example 1: [12,35,1,10,34,1] → Largest=35, Second Largest=34
Example 2: [10,10,10] → -1 (sab same)

Algorithm — Two Traversals

Pass 1: Largest element find karo.
Pass 2: Largest se chhota aur sabse bada element find karo — wahi Second Largest.

int getSecondLargest(vector<int> &arr) { int n = arr.size(); // Pass 1: Largest Element int largest = arr[0]; for (int i = 1; i < n; i++) { if (arr[i] > largest) { largest = arr[i]; } } // Pass 2: Second Largest (largest se chhota, sabse bada) int secondLargest = -1; for (int i = 0; i < n; i++) { if (arr[i] < largest && arr[i] > secondLargest) { secondLargest = arr[i]; } } return secondLargest; }

Problem 5 — GFG: Segregate 0s and 1s

Problem Statement

Array me sirf 0 aur 1 diye gaye hain. Saare 0 ko beginning me aur saare 1 ko ending me arrange karna hai. Relative order maintain karna zaroori nahi hai.

Example: [0,1,0,1,1,0] → [0,0,0,1,1,1]

Approach 1: Brute Force (Temp Array) O(n)

void segregate0and1(vector<int> &arr) { int n = arr.size(); vector<int> temp(n); int index = 0; // Step 1: Saare 0s store karo for (int i = 0; i < n; i++) { if (arr[i] == 0) { temp[index++] = arr[i]; } } // Step 2: Saare 1s store karo for (int i = 0; i < n; i++) { if (arr[i] == 1) { temp[index++] = arr[i]; } } // Step 3: Copy back for (int i = 0; i < n; i++) { arr[i] = temp[i]; } }

Approach 2: Counting Method O(n)

Intuition / Insight

Pehle count karo kitne 0 hain. Phir first countZero positions par 0 fill kar do, baaki sab jagah 1. Extra array ki zaroorat nahi.

void segregate0and1(vector<int> &arr) { int n = arr.size(); int countZero = 0; // Step 1: Count total 0s for (int i = 0; i < n; i++) { if (arr[i] == 0) countZero++; } // Step 2: First countZero positions par 0 for (int i = 0; i < countZero; i++) { arr[i] = 0; } // Step 3: Remaining positions par 1 for (int i = countZero; i < n; i++) { arr[i] = 1; } }

Approach 3: Two Pointers (Optimal) O(n) — Best

Intuition / Insight

Left pointer ko 0 chahiye, Right pointer ko 1 chahiye. Agar dono wrong position par hain to swap kar do. Ye Dutch National Flag pattern ka simplified version hai.

void segregate0and1(vector<int> &arr) { int left = 0; int right = arr.size() - 1; while (left < right) { // Left already correct hai (0 hai) if (arr[left] == 0) { left++; } // Right already correct hai (1 hai) else if (arr[right] == 1) { right--; } // Dono wrong position par hain → swap else { swap(arr[left], arr[right]); left++; right--; } } }

Problem 6 — LeetCode 169: Majority Element

Problem Statement

Majority Element wo element hota hai jo array me n/2 se zyada baar aata hai. LeetCode guarantee karta hai ki Majority Element hamesha exist karega.

Example 1: [3,2,3] → 3 appears 2 times, n/2=1 → 3
Example 2: [2,2,1,1,1,2,2] → 2 appears 4 times, n/2=3 → 2

Approach 1: Brute Force O(n²)

int majorityElement(vector<int>& nums) { int n = nums.size(); for (int i = 0; i < n; i++) { int freq = 0; for (int j = 0; j < n; j++) { if (nums[i] == nums[j]) freq++; } if (freq > n / 2) return nums[i]; } return -1; }

Approach 2: Hashing O(n)

int majorityElement(vector<int>& nums) { int n = nums.size(); unordered_map<int, int> freq; // Step 1: Frequency Count for (int i = 0; i < n; i++) { freq[nums[i]]++; } // Step 2: Find Majority for (int i = 0; i < n; i++) { if (freq[nums[i]] > n / 2) return nums[i]; } return -1; }

Approach 3: Moore's Voting Algorithm (Optimal) O(n) — Best

Intuition / Insight

Socho ek election ho raha hai. Candidate aur Count maintain karo. Agar current element same candidate hai to vote (count++), agar different hai to vote cancel (count--).

Jab count = 0 ho jaye to naya candidate choose karo. Kyunki Majority Element ki frequency n/2 se zyada hai, baaki sab uske votes completely cancel nahi kar sakte. End me jo candidate bachega wahi Majority Element hoga.

int majorityElement(vector<int>& nums) { int candidate = 0; int count = 0; for (int i = 0; i < nums.size(); i++) { // Count 0 → naya candidate if (count == 0) { candidate = nums[i]; } // Same candidate → vote badhao if (nums[i] == candidate) { count++; } // Different candidate → vote ghatao else { count--; } } return candidate; }

Critical Insight

Moore's Voting Algorithm sirf candidate suggest karta hai. Agar problem me guarantee na ho ki Majority Element exist karta hai, to second pass me candidate ki frequency verify karni padti hai. LeetCode 169 me guarantee hai isliye single pass sufficient hai.

Key Takeaways

Takeaway / Summary
  • Rotate by One: Last save karo, right shift karo, first par last rakho — O(n) time, O(1) space.
  • Rotate by K: k = k % n hamesha karo. 3-Reverse optimal hai — O(n) time, O(1) space.
  • Sum of Unique: Bounded constraints pe frequency array best hai — O(n) time, O(1) space.
  • Second Largest: Do linear passes — pehle largest, phir largest se chhota sabse bada.
  • Segregate 0s & 1s: Two Pointer (Dutch National Flag simplified) — single pass, in-place.
  • Majority Element: Moore's Voting Algorithm — candidate aur count maintain karo, votes cancel hote hain.
  • Common Pattern: Two Pointer, Frequency Counting, 3-Reverse — ye teen patterns bahut problems me repeat hote hain.
