Problem 1 : Kth Missing Positive Number [LeetCode 1539]

Problem Statement Given: Ek sorted array of positive integers aur ek integer k.
Task: kth missing positive number return karna hai.
Missing number woh positive integer hai jo array mein present nahi hai.
Examples
InputkOutputMissing Numbers
[2,3,4,7,11]591, 5, 6, 8, 9, 10, 12...
[1,2,3,4]265, 6, 7, 8...
[2]111, 3, 4, 5...
[5,6,7]331, 2, 3, 4, 8...

Core Intuition

Intuition / Insight Array sorted hai, toh har index i pe expected value hona chahiye i + 1.

Jaise [2,3,4,7,11] mein index 0 pe expected 1 hai, lekin actual 2 hai. Matlab 2 - 1 = 1 number missing hai index 0 tak.

Formula: Missing count till index i = arr[i] - (i + 1)

Yeh missing count monotonically increase hota hai — jitna aage jayenge, utne zyada numbers missing honge. Isliye Binary Search directly lag sakta hai.

Approach 1 : Brute Force (Linear Scan)

Workflow / Algorithm
  1. Array ko i = 0 se traverse karo.
  2. Har index i pe missing numbers ka count calculate karo: missing = arr[i] - (i + 1).
  3. Agar missing >= k → kth missing number current element se pehle lie karta hai. Return i + k.
  4. Agar loop end tak koi aisa index nahi mila → kth missing number last element ke baad hai. Return n + k.
// Brute Force: O(n)
class Solution {
public:
    int findKthPositive(vector<int>& arr, int k) {
        int n = arr.size();

        for (int i = 0; i < n; i++) {
            // Index i tak kitne numbers missing hain
            int missing = arr[i] - (i + 1);

            // Kth missing number arr[i] se pehle hai
            if (missing >= k) {
                return i + k;
            }
        }
        // Kth missing number last element ke baad hai
        return n + k;
    }
};
Complexity Analysis
MetricValueReason
TimeO(n)Array ek baar traverse hota hai
SpaceO(1)Extra variables only
Why Not This Approach? Yeh approach sirf array ko ek baar scan karti hai, isliye pehle wale two-pointer se behtar hai. Lekin O(n) time leta hai. Kyunki missing count monotonic hai (kabhi decrease nahi hota), Binary Search se O(log n) mein directly kaam ho jata hai.

Approach 2 : Optimal (Binary Search)

Key Insight Har index i pe missing numbers ka count calculate kar sakte hain: missing = arr[i] - (i + 1).

Hum search karte hain pehla index jahan missing count >= k ho. Uske pehle wale index tak missing count < k hota hai.

Jab loop end hota hai, start us pehle index ko point karta hai jahan se missing count >= k shuru hota hai. Final answer: start + k.

Logic: Agar index start tak start numbers present hain, toh start + k woh number hoga jo k missing numbers ke baad aata hai.
Algorithm Steps
  1. start = 0, end = n - 1 initialize karo.
  2. while (start <= end) loop chalao.
  3. mid = start + (end - start) / 2 calculate karo.
  4. missing = arr[mid] - (mid + 1) calculate karo.
  5. missing < k → Abhi tak k se kam missing hain. Right mein jao: start = mid + 1.
  6. missing >= k → Ya toh exact k ya usse zyada missing hain. Left mein jao: end = mid - 1.
  7. Loop end hone pe return start + k.
// Optimal: Binary Search - O(log n)
class Solution {
public:
    int findKthPositive(vector<int>& arr, int k) {
        int start = 0;
        int end = arr.size() - 1;

        while (start <= end) {
            int mid = start + (end - start) / 2;

            // Missing numbers till index mid
            int missing = arr[mid] - (mid + 1);

            // Abhi tak k se kam missing hain -> right mein jao
            if (missing < k) {
                start = mid + 1;
            }
            // Missing count k ya usse zyada -> left mein jao
            else {
                end = mid - 1;
            }
        }
        return start + k;           // Final formula
    }
};

Edge Cases & Gotchas
  • arr = [1,2,3,4], k = 2: Missing count kabhi k se zyada nahi hoga loop mein. End mein start = 4 (array size), answer = 4 + 2 = 6. Correct!
  • arr = [2], k = 1: missing = 2 - 1 = 1 >= 1 → end = -1. start = 0. Answer = 0 + 1 = 1. Correct!
  • Formula samajhna zaroori: start + k kyun? Kyunki start index tak start numbers present hain. k missing numbers ke baad jo number aata hai woh start + k hota hai.
  • Missing count monotonic hai: Isliye binary search kaam karta hai. Agar array unsorted hota, toh brute force hi chahiye hota.
Best Practice / Optimization
  • missing = arr[mid] - (mid + 1) formula yaad rakho — yeh sabse important hai.
  • Loop condition start <= end use karo — classic lower bound style.
  • End mein start + k return karo — yeh formula proof se aata hai, rote mat learn karo.

Problem 2 : Allocate Minimum Number of Pages [GFG]

Problem Statement Given: Ek array books[] jisme books[i] = ith book ke pages. Aur ek integer students.
Task: Books ko students mein allocate karna hai such that:
• Har student ko at least one book mile.
• Books contiguous order mein allocate honge.
• Koi book divide nahi ho sakti.
Goal: Maximum pages assigned to any student ko minimize karna hai.
Impossible case: Agar students > books.size(), return -1.
Examples
BooksStudentsOutputAllocation
[12,34,67,90]2113[12+34+67=113], [90]
[15,17,20]5-1Books < students, impossible
[10,20,30,40]260[10+20+30=60], [40]
[10,20,30,40]360[10+20=30], [30], [40] → max=60... wait let me verify

Core Intuition

Intuition / Insight Yeh problem directly array mein search nahi kar rahi — yeh "answer space" mein search kar rahi hai.

Question: Kya maximum X pages per student allowed ho, toh kya saari books allocate ho sakti hain?

Agar X bada hai, toh allocation easy hai (kam students chahiye). Agar X chota hai, toh allocation mushkil hai (zyada students chahiye).

Yeh monotonic property hai: Impossible → Impossible → Possible → Possible. Isliye Binary Search first possible answer dhoondh sakta hai.

Search Space:
• Minimum possible answer: Sabse badi book (kyunki har student ko kam se kam woh book toh milegi hi).
• Maximum possible answer: Saari books ka sum (ek hi student ko sab mil jaye).
Monotonic Property Visualization
40 50 60 70 80 90 100
Impossible → First Possible (Answer) → Possible
Binary Search finds the boundary between Impossible and Possible

Approach 1 : Brute Force (Linear Search on Answer)

Workflow / Algorithm
  1. Minimum answer = largest book se start karo.
  2. Maximum answer = total sum of all pages tak jao.
  3. Har value X ke liye check karo: kya X max pages mein allocation possible hai?
  4. First valid X return karo.
Complexity Analysis
MetricValueReason
TimeO(n x sum)Har possible X ke liye O(n) check
SpaceO(1)Extra variables only
Why Not Brute Force? Sum of pages bahut bada ho sakta hai (10^9+ range). Linear search on answer space TLE de degi. Binary Search se O(log(sum)) iterations mein kaam ho jata hai.

Approach 2 : Optimal (Binary Search on Answer)

Key Insight Two-step approach:
Step 1: possible(X) function — greedy approach se check karo ki X max pages limit mein saari books students count mein allocate ho sakti hain ya nahi.

Step 2: Binary Search on answer space [maxBook, totalSum] to find minimum valid X.

Greedy Allocation: Har student ko jitni books de sakte hain bina X cross kiye, utni do. Agar next book se X cross ho raha hai, toh naya student allocate karo.
Algorithm Steps
  1. Impossible check: Agar students > books.size(), return -1.
  2. start = max_element(books), end = sum(books).
  3. answer = end initialize karo.
  4. while (start <= end):
    • mid = start + (end - start) / 2.
    • possible(books, students, mid) check karo.
    • Agar possible → answer = mid store karo, end = mid - 1 (chota answer dhoondho).
    • Agar impossible → start = mid + 1 (bada answer chahiye).
  5. Return answer.
// Optimal: Binary Search on Answer - O(n x log(sum))
class Solution {
public:
    // Greedy check: kya 'maxPages' limit mein allocation possible hai?
    bool possible(vector<int>& books, int students, long long maxPages) {
        int requiredStudents = 1;
        long long currentPages = 0;

        for (int pages : books) {
            // Same student ko aur books do
            if (currentPages + pages <= maxPages) {
                currentPages += pages;
            }
            // Naya student allocate karo
            else {
                requiredStudents++;
                currentPages = pages;
            }
        }
        return requiredStudents <= students;
    }

    int findPages(vector<int>& books, int students) {
        int n = books.size();

        if (students > n)
            return -1;           // Impossible case

        long long start = *max_element(books.begin(), books.end());
        long long end = accumulate(books.begin(), books.end(), 0LL);
        long long answer = end;

        while (start <= end) {
            long long mid = start + (end - start) / 2;

            // Allocation possible hai -> aur chota answer try karo
            if (possible(books, students, mid)) {
                answer = mid;
                end = mid - 1;
            }
            // Allocation impossible -> limit badhao
            else {
                start = mid + 1;
            }
        }
        return (int)answer;
    }
};

Edge Cases & Gotchas
  • students > books.size(): Directly return -1. Har student ko kam se kam ek book chahiye.
  • Single book: [10], students=1 → start=10, end=10, answer=10.
  • All same pages: [10,10,10,10], students=2 → Optimal: [10+10=20], [10+10=20] → 20.
  • Large numbers: long long use karo sum ke liye — int overflow ho sakta hai.
  • Greedy proof: Greedy allocation (jitna de sakte ho utna do) hamesha optimal hota hai kyunki hum minimize kar rahe hain max pages per student.
Interview Tips / Follow-up Questions
  • Q: Yeh problem "Painter's Partition" se kaise related hai?
    A: Same pattern! Painters = students, boards = books, time = pages. Dono mein same binary search on answer lagta hai.
  • Q: Agar books non-contiguous allocate ho sakti hain toh?
    A: Problem completely different ho jayegi. Contiguous constraint hi monotonic property deti hai.
  • Q: possible() function ko optimize kar sakte hain?
    A: O(n) se better nahi kar sakte kyunki saari books check karni padti hain.
  • Q: Agar students = 1 ho toh?
    A: Answer = sum of all pages. Binary Search mein start=end=sum, directly return.

Kth Missing Positive (LC 1539)
• Missing till index i = arr[i] - (i + 1)
• Monotonic property → Binary Search
• Find first index where missing >= k
• Answer = start + k
• Time: O(log n) | Space: O(1)
Book Allocation (GFG)
• Binary Search on Answer Space
• Search range: [maxBook, totalSum]
• Greedy possible() check: O(n)
• Valid? → store, search left
• Invalid? → search right
• Time: O(n x log(sum)) | Space: O(1)
Binary Search on Answer Pattern
• Jab direct array mein answer nahi mile
• Answer ka range pata ho
• Monotonic feasibility property ho
• Helper function se validity check
• Examples: Sqrt(x), Book Allocation, Painter's, Ship Capacity
Common Mistakes (Red Flags)
• Missing formula galat: arr[i]-(i+1) hi hai
• Book alloc mein students > books: return -1
• Sum mein int overflow: long long use karo
• Greedy proof bhoolna: contiguous = greedy optimal
• Answer formula: Kth missing = start + k
