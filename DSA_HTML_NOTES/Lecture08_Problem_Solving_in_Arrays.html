Lecture 08: Problem Solving in Arrays

Is lecture mein hum arrays ke upar 5 important problems solve karenge — Array ko Function mein pass karna, Array Search, Reverse Array, Check Sorted, aur Missing Number. Har problem ko multiple approaches se samajhenge taaki aap har situation mein best solution choose kar sako.

Passing Array to a Function

Intuition Jab aap array ko function mein pass karte ho, toh C++ mein wo Pass by Reference hota hai — matlab original array ka address jaata hai, copy nahi. Isliye agar function mein array modify karte ho, toh original array bhi change ho jata hai.
#include <iostream> using namespace std; // Array ko function mein pass karna void printValue(int a[], int n) { for(int i = 0; i < n; i++) { cout << a[i] << " "; } cout << endl; // Original array modify kar rahe hain a[0] = 90; } int main() { int arr[5] = {10, 20, 30, 40, 50}; printValue(arr, 5); // Array pass kiya cout << "arr[0] after function: " << arr[0] << endl; // Output: 90 (Original change ho gaya!) return 0; }
Important Array function mein hamesha Pass by Reference hota hai — chahe aap & lagao ya na lagao. Ye C++ ka default behavior hai arrays ke liye. Isliye function mein array modify karne se original array bhi change hota hai.

Problem 1: Array Search (Linear Search)

Intuition Array ke har element ko ek-ek karke check karo. Target element mil jaye toh uska index return karo. Agar pura array check karne ke baad bhi na mile, toh -1 return karo (not found).

Problem Statement

Input: arr = [3, 2, 8, 4], x = 3
Output: 2 (kyunki 3 index 2 pe hai)

#include <iostream> using namespace std; int search(int arr[], int n, int x) { for(int i = 0; i < n; i++) { if(arr[i] == x) { return i; // Element mil gaya, index return karo } } return -1; // Element nahi mila } int main() { int arr[4] = {3, 2, 8, 4}; int x = 3; int result = search(arr, 4, x); if(result != -1) { cout << "Element found at index: " << result << endl; } else { cout << "Element not found" << endl; } return 0; }

Performance Linear Search ki Time Complexity: O(n) — worst case mein pura array traverse karna padta hai. Space Complexity: O(1) — extra space nahi lagti.

Problem 2: Reverse an Array

Intuition Array ke pehle element ko last se swap karo, dusre ko second last se, aur aage badhte jao. Middle tak pahunchne pe array reverse ho jayega. Formula: arr[i] swap with arr[n-1-i].

#include <iostream> using namespace std; void reverseArray(int arr[], int n) { // Sirf n/2 tak chalana hai — warna dobara swap ho jayega for(int i = 0; i < n/2; i++) { // arr[i] ko arr[n-1-i] se swap karo int temp = arr[i]; arr[i] = arr[n-1-i]; arr[n-1-i] = temp; } } int main() { int arr[5] = {10, 15, 18, 22, 28}; cout << "Before: "; for(int i = 0; i < 5; i++) cout << arr[i] << " "; cout << endl; reverseArray(arr, 5); cout << "After: "; for(int i = 0; i < 5; i++) cout << arr[i] << " "; cout << endl; return 0; }

Problem 3: Check if Array is Sorted

Intuition Array sorted hai agar har element apne pehle wale element se bada ya barabar ho. Matlab arr[i] >= arr[i-1] har valid i ke liye true hona chahiye. Agar koi bhi element chhota mila, toh array sorted nahi hai.

#include <iostream> using namespace std; bool isSorted(int arr[], int n) { // Index 1 se start karo aur previous element se compare karo for(int i = 1; i < n; i++) { if(arr[i-1] > arr[i]) { return false; // Previous element bada hai — not sorted } } return true; // Sab theek hai — sorted hai } int main() { int arr1[5] = {10, 20, 30, 40, 50}; int arr2[5] = {50, 40, 30, 20, 10}; cout << "arr1 sorted? " << (isSorted(arr1, 5) ? "YES" : "NO") << endl; cout << "arr2 sorted? " << (isSorted(arr2, 5) ? "YES" : "NO") << endl; return 0; }

Problem 4: Missing Number in Array

Intuition Array mein 1 se n tak ke numbers hain, lekin ek number missing hai. Us missing number ko find karna hai. Is problem ko 3 tarike se solve kar sakte hain — Brute Force, Sum Formula, aur XOR.

Problem Statement

Input: arr = [3, 1, 2, 5] (1 to 5 mein se 4 missing hai)
Output: 4

Approach 1: Brute Force Approach 2: Sum Formula Approach 3: XOR

Approach 1: Brute Force (O(n²))

int missingNumber(int arr[], int n) { int size = n + 1; // Actual size = n+1 (ek missing hai) for(int i = 1; i <= size; i++) { bool found = false; for(int j = 0; j < n; j++) { if(arr[j] == i) { found = true; break; } } if(!found) { return i; // Ye number missing hai } } return -1; }
Performance Brute Force ki Time Complexity: O(n²) — nested loop hai. TLE (Time Limit Exceeded) ho sakta hai bade inputs pe. Interview mein ye approach batana chalega, lekin optimized version bhi batana zaroori hai.

Approach 2: Sum Formula (O(n)) ⭐ Recommended

Intuition Sum of first n natural numbers ka formula: n × (n+1) / 2. Agar array ke elements ka sum nikalo aur expected sum se minus karo, toh missing number mil jayega.

Missing Number = Expected Sum - Actual Sum
int missingNumber(int arr[], int n) { int size = n + 1; // Kyunki ek number missing hai // Step 1: Expected sum nikalo (1 se size tak) int expectedSum = size * (size + 1) / 2; // Step 2: Actual sum nikalo int actualSum = 0; for(int i = 0; i < n; i++) { actualSum += arr[i]; } // Step 3: Difference hi missing number hai return expectedSum - actualSum; }
Workflow — arr = [3, 1, 2, 5], n = 4

Performance Sum Formula ki Time Complexity: O(n) — single loop. Space Complexity: O(1). Ye approach interviews mein sabse zyada prefer ki jati hai.

Approach 3: XOR (O(n)) — Bit Manipulation

Intuition XOR ka ek property hai: a ^ a = 0 aur a ^ 0 = a. Agar saare numbers ko XOR karo (1 se n tak) aur saare array elements ko bhi XOR karo, toh same numbers cancel ho jayenge aur sirf missing number bachega.
int missingNumberXOR(int arr[], int n) { int size = n + 1; int ans = 0; // Step 1: 1 se size tak XOR karo for(int i = 1; i <= size; i++) { ans = ans ^ i; } // Step 2: Array ke saare elements XOR karo for(int i = 0; i < n; i++) { ans = ans ^ arr[i]; } return ans; // Missing number mil gaya }
Workflow — XOR Property

Edge Case XOR approach integer overflow se bachata hai. Sum formula mein bade numbers pe overflow ho sakta hai (int ki limit cross). XOR mein koi overflow nahi hota. Lekin XOR approach thoda complex hai samajhne mein.

Problem 5: Vector Basics (Dynamic Array)

Intuition Normal array ki problem hai ki uska size fix hota hai. Vector ek dynamic array hai jo apne aap size badha sakta hai. push_back() se element add karo, pop_back() se last element remove karo.
#include <iostream> #include <vector> // Vector library include karna zaroori using namespace std; int main() { // Vector declare karna vector<int> arr; // Elements add karna (size automatically badhega) arr.push_back(10); // arr = [10] arr.push_back(20); // arr = [10, 20] arr.push_back(30); // arr = [10, 20, 30] // Last element remove karna arr.pop_back(); // arr = [10, 20], last element (30) remove // Size check karna cout << "Size: " << arr.size() << endl; // 2 // Traverse karna (same as array) for(int i = 0; i < arr.size(); i++) { cout << arr[i] << " "; } return 0; }

Key Takeaways

Takeaway
  • Array in Function: Hamesha Pass by Reference hota hai. Function mein modify karne se original array change hota hai.
  • Linear Search: Har element check karo. O(n) time. Element milne pe index return, nahi toh -1.
  • Reverse Array: arr[i] ko arr[n-1-i] se swap karo. Loop n/2 tak chalao.
  • Check Sorted: Har element apne previous se compare karo. arr[i-1] > arr[i] ho toh not sorted.
  • Missing Number (Brute): Nested loop se har number check karo. O(n²) — TLE risk.
  • Missing Number (Sum): Expected Sum - Actual Sum. O(n) — Best for interviews.
  • Missing Number (XOR): XOR properties use karo. O(n) — Overflow safe, lekin complex.
  • Vector: Dynamic array hai. push_back(), pop_back(), size() — main operations.

Interview Tip Missing Number problem mein hamesha Sum Formula approach batana pehle — ye sabse intuitive hai. Phir XOR approach batana as an alternative. Brute Force sirf tab batana jab interviewer specifically puche. "Expected Sum minus Actual Sum" — ye line yaad rakho, interview clear ho jayega.
Key Insight: Array problems solve karne ka golden rule: Pehle brute force socho, phir optimize karo. Har problem ke liye multiple approaches hote hain — basic se lekar advanced tak.
