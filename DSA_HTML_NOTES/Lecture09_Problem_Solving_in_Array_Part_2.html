Problem 1: Rotate Array by k Positions

LeetCode 189 | GFG: Rotate Array
Difficulty: Medium | Array Manipulation | In-Place

Problem Statement:

Given an integer array nums, rotate the array to the right by k steps, where k is non-negative.

Constraints:

  • 1 <= nums.length <= 105
  • -231 <= nums[i] <= 231 - 1
  • 0 <= k <= 105
// Example 1 Input: nums = [1, 2, 3, 4, 5, 6, 7], k = 3 Output: [5, 6, 7, 1, 2, 3, 4] // Explanation: rotate 1 step to the right: [7,1,2,3,4,5,6] // rotate 2 steps to the right: [6,7,1,2,3,4,5] // rotate 3 steps to the right: [5,6,7,1,2,3,4]
// Example 2 Input: nums = [-1, -100, 3, 99], k = 2 Output: [3, 99, -1, -100] // Explanation: rotate 1 step to the right: [99,-1,-100,3] // rotate 2 steps to the right: [3,99,-1,-100]
// Example 3 (k > n) Input: nums = [1, 2], k = 3 Output: [2, 1] // Explanation: k = 3 % 2 = 1 effective rotation
Pattern Array Manipulation — Right Rotation, Reversal Algorithm, In-Place

Introduction

Is problem mein array ke elements ko right side se k positions rotate karna hai. Right rotation ka matlab hai ki last ke k elements front mein aa jayenge. Example: [1, 2, 3, 4, 5] ko k=2 se rotate karne par [4, 5, 1, 2, 3] ban jayega. Is problem ko solve karne ke teen approaches hain.

Key Insight: Agar k > n, toh extra rotations ka koi fayda nahi. Effective rotations k = k % n honge. Example: n=5, k=7 ke liye sirf 2 rotations kaam aayengi.

Approach 1: Brute Force (Rotate by One, k Times)

Basic

Ek baar rotate karna pata hai — last element ko save karo, baaki elements ko right shift karo, aur saved element ko front mein rakho. Ye process k baar repeat karo.

Workflow Step 1: k = k % n (effective rotations).
Step 2: Outer loop i = 0 to k-1.
Step 3: Save last element.
Step 4: Right shift all elements by one.
Step 5: Put saved element at index 0.
// Approach 1: Brute Force -- Rotate by One, k Times class Solution { public: void rotate(vector<int>& nums, int k) { int n = nums.size(); // Effective rotations nikal lo k = k % n; // Rotate by One, k times for (int i = 0; i < k; i++) { // Last element ko save kar lo int last = nums[n - 1]; // Saare elements ko ek position right shift karo for (int j = n - 1; j > 0; j--) { nums[j] = nums[j - 1]; } // Saved last element ko first position par rakh do nums[0] = last; } } };
TimeSpaceDrawback
O(n * k)O(1)Nested loop ki wajah se very slow for large k.

Approach 2: Using Temporary Array

Intermediate

Har element ki new position calculate karo: newIndex = (currentIndex + k) % n. Saare elements ko temporary array mein unki new positions pe rakho, phir temporary array ko original mein copy karo.

Workflow Formula: newPosition = (i + k) % n.
Step 1: Create temp array of size n.
Step 2: temp[(i + k) % n] = nums[i] for all i.
Step 3: Copy temp back to nums.
// Approach 2: Using Temporary Array class Solution { public: void rotate(vector<int>& nums, int k) { int n = nums.size(); k = k % n; vector<int> temp(n); // Har element ko uski new position par rakho for (int i = 0; i < n; i++) { temp[(i + k) % n] = nums[i]; } // Temp array ko original array me copy karo for (int i = 0; i < n; i++) { nums[i] = temp[i]; } } };
TimeSpaceDrawback
O(n)O(n)Extra array required, not space optimal.

Approach 3: Reverse Algorithm (Optimal)

Advanced

Ye approach three reversals se kaam karti hai. Pehle poore array ko reverse karo, phir first k elements ko reverse karo, aur last me remaining elements ko reverse karo. Isse array right rotate ho jata hai.

Workflow Initial: [1, 2, 3, 4, 5], k = 2
Step 1: Reverse all → [5, 4, 3, 2, 1]
Step 2: Reverse first k → [4, 5, 3, 2, 1]
Step 3: Reverse remaining → [4, 5, 1, 2, 3]
Result: Right rotated by k positions.
// Approach 3: Reverse Algorithm (Optimal) class Solution { public: void rotate(vector<int>& nums, int k) { int n = nums.size(); k = k % n; // Step 1: Reverse complete array reverse(nums.begin(), nums.end()); // Step 2: Reverse first k elements reverse(nums.begin(), nums.begin() + k); // Step 3: Reverse remaining elements reverse(nums.begin() + k, nums.end()); } };

Custom Reverse Function Implementation

Agar built-in reverse function use nahi karna, toh custom Two Pointer reverse function bhi likh sakte hain:

// Custom Reverse Function class Solution { public: void reverseArray(vector<int>& nums, int left, int right) { // Jab tak Left aur Right cross na kare while (left < right) { int temp = nums[left]; nums[left] = nums[right]; nums[right] = temp; left++; right--; } } void rotate(vector<int>& nums, int k) { int n = nums.size(); k = k % n; // Step 1: Poora array reverse karo reverseArray(nums, 0, n - 1); // Step 2: First k elements reverse karo reverseArray(nums, 0, k - 1); // Step 3: Remaining elements reverse karo reverseArray(nums, k, n - 1); } };
TimeSpaceAdvantage
O(n)O(1)In-place, no extra array, most efficient.
Interview Tip Reverse Algorithm interview mein most expected approach hai. Iska logic samajhna zaroori hai: reverse all → reverse first k → reverse remaining. Is pattern ko yaad rakho kyunki ye String Rotation problems mein bhi use hota hai. Custom reverse function likhna seekho taaki built-in functions pe dependent na raho.
Key Takeaways
  • k = k % n hamesha karo taaki extra rotations avoid ho.
  • Brute Force O(n * k) time leta hai, avoid karo.
  • Temporary Array O(n) time lekin O(n) space leta hai.
  • Reverse Algorithm O(n) time, O(1) space — optimal solution.
  • Custom reverse function Two Pointer technique se likh sakte hain.

Problem 2: Cyclically Rotate Array by One

GFG: Cyclically Rotate an Array by One
Difficulty: Easy | Array Manipulation | Right Rotation

Problem Statement:

Given an array, rotate the array by one position in clockwise direction (right rotation by 1).

Constraints:

  • 1 <= arr.size() <= 104
  • 0 <= arr[i] <= 105
// Example 1 Input: arr = [1, 2, 3, 4, 5] Output: [5, 1, 2, 3, 4] // Explanation: Last element (5) moves to front, rest shift right
// Example 2 Input: arr = [9, 8, 7, 6, 4, 2, 1, 3] Output: [3, 9, 8, 7, 6, 4, 2, 1] // Explanation: Last element (3) moves to front
// Example 3 (Single Element) Input: arr = [42] Output: [42] // Explanation: Single element, no change
Pattern Array Manipulation — Right Rotation by 1, Single Pass

Introduction

Ye problem Rotate Array by k ka special case hai jahan k = 1. Sirf ek position right rotate karna hai. Last element front mein aa jayega aur baaki sab ek step right shift honge.

How It Works Internally

  1. Last element ko save karo.
  2. Array ke elements ko right se shift karo.
  3. Saved element ko index 0 pe rakh do.
Workflow Initial: [1, 2, 3, 4, 5]
Save: last = 5
Shift: [1, 1, 2, 3, 4] → right se shift karte hue overwrite hota hai
Place: [5, 1, 2, 3, 4]
Result: Array right rotated by 1.

Code Implementation

// GFG: Cyclically Rotate an Array by One class Solution { public: void rotate(vector<int> &arr) { int n = arr.size(); int last = arr[n - 1]; // Right se shift karo taki data overwrite na ho for (int i = n - 1; i > 0; i--) { arr[i] = arr[i - 1]; } // Saved last element ko front me rakh do arr[0] = last; } };
TimeSpaceNote
O(n)O(1)Single pass, right-to-left shift to avoid overwrite.
Key Takeaways
  • Right-to-left shift karo taaki data overwrite na ho.
  • Left-to-right shift karne se data loss ho jayega.
  • Time O(n), Space O(1).

Problem 3: Sum of Unique Elements

LeetCode 1748: Sum of Unique Elements
Difficulty: Easy | Frequency Counting | Hashing

Problem Statement:

You are given an integer array nums. The unique elements of an array are the elements that appear exactly once in the array. Return the sum of all the unique elements of nums.

Constraints:

  • 1 <= nums.length <= 100
  • 1 <= nums[i] <= 100
// Example 1 Input: nums = [1, 2, 3, 2] Output: 4 // Explanation: Unique elements are 1 and 3. Sum = 1 + 3 = 4
// Example 2 Input: nums = [1, 1, 1, 1, 1] Output: 0 // Explanation: No unique elements. Sum = 0
// Example 3 Input: nums = [1, 2, 3, 4, 5] Output: 15 // Explanation: All elements are unique. Sum = 1+2+3+4+5 = 15
Pattern Frequency Counting — Hashing, Brute Force, Array Constraints

Introduction

Is problem mein array ke un elements ka sum nikalna hai jo sirf ek baar appear hote hain. Duplicates ko ignore karna hai. Iske liye do approaches hain: Brute Force aur Hashing.

Approach 1: Brute Force (Nested Loops)

Basic

Har element ki frequency poore array mein count karo. Agar frequency 1 hai, toh usse sum mein add karo.

// Approach 1: Brute Force class Solution { public: int sumOfUnique(vector<int>& nums) { int n = nums.size(); int sum = 0; // Har element ke liye frequency count karo for (int i = 0; i < n; i++) { int freq = 0; for (int j = 0; j < n; j++) { if (nums[i] == nums[j]) { freq++; } } // Agar sirf ek baar aaya hai to answer me add kar do if (freq == 1) { sum += nums[i]; } } return sum; } };
TimeSpaceDrawback
O(n²)O(1)Nested loops, very slow for large arrays.

Approach 2: Hashing (Frequency Array)

Intermediate

Since LeetCode 1748 mein values 1 se 100 ke beech hain, hum fixed-size frequency array use kar sakte hain. Pehle frequency count karo, phir unique elements ka sum nikalo.

// Approach 2: Hashing (Frequency Array) class Solution { public: int sumOfUnique(vector<int>& nums) { vector<int> freq(101, 0); // Frequency count karo for (int i = 0; i < nums.size(); i++) { freq[nums[i]]++; } int sum = 0; // Unique elements ka sum nikalo for (int i = 0; i < nums.size(); i++) { if (freq[nums[i]] == 1) { sum += nums[i]; } } return sum; } };
TimeSpaceAdvantage
O(n)O(1)Fixed size freq array, two linear passes.
Interview Tip Agar constraints nahi pata hain, toh unordered_map use karo. Lekin jab range known ho (jaise 1-100), toh array-based hashing faster aur simpler hota hai. Interviewer ko batana ki aap constraints ke hisaab se approach optimize karte ho.
Key Takeaways
  • Brute Force O(n²) time leta hai, avoid karo.
  • Hashing se time O(n) mein optimize hota hai.
  • Known range pe array-based frequency count O(1) space mein kaam karta hai.
  • unordered_map generic solution hai jab range unknown ho.

Problem 4: Second Largest Element

GFG: Second Largest Element
Difficulty: Easy | Array Traversal | Without Sorting

Problem Statement:

Given an array of positive integers arr[], return the second largest element from the array. If the second largest element does not exist then return -1.

Constraints:

  • 1 <= arr.size() <= 105
  • 1 <= arr[i] <= 105
// Example 1 Input: arr = [12, 35, 1, 10, 34, 1] Output: 34 // Explanation: Largest = 35, Second Largest = 34
// Example 2 Input: arr = [10, 5, 10] Output: 5 // Explanation: Largest = 10, Second Largest = 5
// Example 3 (All Same) Input: arr = [10, 10, 10] Output: -1 // Explanation: No second largest element exists
Pattern Array Traversal — Two Passes, Without Sorting, Linear Time

Introduction

Array mein second largest element dhoondhna hai bina sort kiye. Iske liye do traversals use karte hain: pehle largest find karo, phir largest se chhota aur sabse bada element find karo.

How It Works Internally

  1. Pass 1: Array traverse karke largest element find karo.
  2. Pass 2: Dubara array traverse karo. Largest se chhota aur sabse bada element second largest hoga.
Workflow Pass 1: largest = max(arr[0..n-1]).
Pass 2: secondLargest = max(arr[i] where arr[i] < largest).
Return: secondLargest. Agar nahi mila toh -1.

Code Implementation

// GFG: Second Largest Element class Solution { public: int getSecondLargest(vector<int> &arr) { int n = arr.size(); int largest = arr[0]; // Step 1: Largest Find Karo for (int i = 1; i < n; i++) { if (arr[i] > largest) { largest = arr[i]; } } int secondLargest = -1; // Step 2: Second Largest Find Karo for (int i = 0; i < n; i++) { if (arr[i] < largest && arr[i] > secondLargest) { secondLargest = arr[i]; } } return secondLargest; } };
TimeSpaceNote
O(n)O(1)Two linear passes, no sorting needed.
Best Practice Single pass mein bhi solve kiya ja sakta hai: largest aur secondLargest dono ko ek hi loop mein update karo. Lekin two-pass approach cleaner aur easier to understand hai. Interviews mein pehle two-pass batana, phir single-pass optimization batana.
Key Takeaways
  • Two traversals mein second largest mil jata hai bina sort kiye.
  • Time O(n), Space O(1).
  • Single pass optimization possible hai lekin readability kam hoti hai.
  • All duplicates case handle karo: secondLargest -1 return hoga.

Problem 5: Segregate 0s and 1s

GFG: Segregate 0s and 1s
Difficulty: Easy | Two Pointers | In-Place Partition

Problem Statement:

Given an array of length n consisting of only 0s and 1s in random order, modify the array in-place to segregate 0s on the left side and 1s on the right side of the array.

Constraints:

  • 1 <= arr.size() <= 105
  • 0 <= arr[i] <= 1
// Example 1 Input: arr = [0, 1, 0, 1, 1, 0, 0, 1] Output: [0, 0, 0, 0, 1, 1, 1, 1] // Explanation: All 0s moved to left, all 1s moved to right
// Example 2 Input: arr = [1, 1, 1, 0, 0, 0] Output: [0, 0, 0, 1, 1, 1]
// Example 3 (Already Sorted) Input: arr = [0, 0, 0, 0] Output: [0, 0, 0, 0]
Pattern Two Pointers — Dutch National Flag Variant, In-Place Partition

Introduction

Array mein sirf 0s aur 1s hain. Humein saare 0s ko left side aur saare 1s ko right side mein arrange karna hai. Is problem ko teen approaches se solve kiya ja sakta hai.

Approach 1: Brute Force (Temporary Array)

Basic

Ek temporary array banayo. Pehle saare 0s store karo, phir saare 1s store karo. Last mein temporary array ko original mein copy karo.

// Approach 1: Brute Force (Temporary Array) class Solution { public: void segregate0and1(vector<int> &arr) { int n = arr.size(); vector<int> temp(n); int index = 0; // Step 1: Pehle saare 0 store karo for (int i = 0; i < n; i++) { if (arr[i] == 0) { temp[index] = arr[i]; index++; } } // Step 2: Fir saare 1 store karo for (int i = 0; i < n; i++) { if (arr[i] == 1) { temp[index] = arr[i]; index++; } } // Step 3: Temporary array ko original array me copy karo for (int i = 0; i < n; i++) { arr[i] = temp[i]; } } };
TimeSpaceDrawback
O(n)O(n)Extra array required, not space optimal.

Approach 2: Counting Method

Intermediate

Pehle total number of 0s count karo. Fir array ke starting mein utne 0s fill karo aur baaki positions pe 1s fill karo.

// Approach 2: Counting Method class Solution { public: void segregate0and1(vector<int> &arr) { int n = arr.size(); int countZero = 0; // Step 1: Count total number of 0s for (int i = 0; i < n; i++) { if (arr[i] == 0) { countZero++; } } // Step 2: First countZero positions par 0 fill karo for (int i = 0; i < countZero; i++) { arr[i] = 0; } // Step 3: Remaining positions par 1 fill karo for (int i = countZero; i < n; i++) { arr[i] = 1; } } };
TimeSpaceAdvantage
O(n)O(1)In-place, no extra array, three linear passes.

Approach 3: Two Pointers (Optimal)

Advanced

Left pointer ko 0 chahiye, Right pointer ko 1 chahiye. Agar dono apni correct position par hain toh move karo. Agar dono wrong position par hain toh swap karo.

Workflow Left wants 0, Right wants 1.
Case 1: arr[left] == 0 → left++ (correct).
Case 2: arr[right] == 1 → right-- (correct).
Case 3: Both wrong → swap(arr[left], arr[right]), then move both.
// Approach 3: Two Pointers (Optimal) class Solution { public: void segregate0and1(vector<int> &arr) { int left = 0; int right = arr.size() - 1; while (left < right) { // Left already correct hai if (arr[left] == 0) { left++; } // Right already correct hai else if (arr[right] == 1) { right--; } // Dono wrong position par hai else { swap(arr[left], arr[right]); left++; right--; } } } };
TimeSpaceAdvantage
O(n)O(1)Single pass, in-place, most elegant solution.
Interview Tip Two Pointer approach Dutch National Flag problem ka simplified version hai. Is pattern ko yaad rakho kyunki ye sort colors (0, 1, 2) aur partition problems mein bhi use hota hai. Interviewer ko pehle counting approach batana, phir Two Pointer optimization batana.
Key Takeaways
  • Brute Force temporary array se O(n) space leta hai.
  • Counting method O(n) time, O(1) space mein kaam karta hai.
  • Two Pointer single pass mein solve karta hai, most elegant.
  • Ye pattern Sort Colors (0, 1, 2) problem mein extend hota hai.

Problem 6: Majority Element

LeetCode 169: Majority Element
Difficulty: Easy | Boyer-Moore Voting | Hashing

Problem Statement:

Given an array nums of size n, return the majority element. The majority element is the element that appears more than n/2 times. You may assume that the majority element always exists in the array.

Constraints:

  • n == nums.length
  • 1 <= n <= 5 * 104
  • -109 <= nums[i] <= 109
// Example 1 Input: nums = [3, 2, 3] Output: 3 // Explanation: 3 appears 2 times out of 3. 2 > 3/2
// Example 2 Input: nums = [2, 2, 1, 1, 1, 2, 2] Output: 2 // Explanation: 2 appears 4 times out of 7. 4 > 7/2
// Example 3 (Single Element) Input: nums = [1] Output: 1
Pattern Frequency / Voting — Moore's Voting Algorithm, Hashing, Brute Force

Introduction

Majority Element wo element hai jo array mein more than n/2 times appear karta hai. Is problem ko solve karne ke teen approaches hain: Brute Force, Hashing, aur Moore's Voting Algorithm.

Approach 1: Brute Force

Basic

Har element ki frequency poore array mein count karo. Agar kisi element ki frequency n/2 se zyada hui, toh wahi Majority Element hai.

// Approach 1: Brute Force class Solution { public: int majorityElement(vector<int>& nums) { int n = nums.size(); for (int i = 0; i < n; i++) { int freq = 0; for (int j = 0; j < n; j++) { if (nums[i] == nums[j]) { freq++; } } // Majority Element mil gaya if (freq > n / 2) { return nums[i]; } } return -1; } };
TimeSpaceDrawback
O(n²)O(1)Nested loops, very slow for large arrays.

Approach 2: Hashing

Intermediate

Hash Map mein har element ki frequency store karo. Phir traverse karke wo element dhoondo jiski frequency n/2 se zyada hai.

// Approach 2: Hashing class Solution { public: int majorityElement(vector<int>& nums) { int n = nums.size(); unordered_map<int, int> freq; // Step 1: Frequency Count Karo for (int i = 0; i < n; i++) { freq[nums[i]]++; } // Step 2: Majority Element Find Karo for (int i = 0; i < n; i++) { if (freq[nums[i]] > n / 2) { return nums[i]; } } return -1; } };
TimeSpaceDrawback
O(n)O(n)Extra space for hash map required.

Approach 3: Moore's Voting Algorithm (Optimal)

Advanced

Ye algorithm ek candidate aur count variable maintain karta hai. Count 0 hone par current element ko naya candidate bana diya jata hai. Same element milne par count badhaya jata hai, different element milne par count ghata diya jata hai. Final candidate hi majority element hota hai.

Key Insight: Moore's Voting Algorithm is based on the fact that if a majority element exists, its count will survive even after canceling out all other elements. Majority element ka count > n/2 hone ki wajah se baaki sab elements milke bhi usse cancel nahi kar sakte.
Workflow Initialize: candidate = 0, count = 0.
Step 1: If count == 0, current element = new candidate.
Step 2: If current == candidate, count++.
Step 3: If current != candidate, count--.
Result: Final candidate is the Majority Element.
// Approach 3: Moore's Voting Algorithm (Optimal) class Solution { public: int majorityElement(vector<int>& nums) { int candidate = 0; int count = 0; // Poore array ko traverse karo for (int i = 0; i < nums.size(); i++) { // Agar candidate ke saare votes cancel ho gaye, // to current element ko naya candidate bana do if (count == 0) { candidate = nums[i]; } // Same candidate mila, to support badha do if (nums[i] == candidate) { count++; } // Different candidate mila, to ek support cancel ho jayega else { count--; } } // Majority Element return karo return candidate; } };
TimeSpaceAdvantage
O(n)O(1)Single pass, no extra space, most elegant.
Interview Tip Moore's Voting Algorithm interview mein must-know hai. Iska logic clear hona chahiye: count == 0 hone par new candidate select karo, same element se count badhao, different se ghatao. Ye algorithm sirf majority element ke liye nahi, n/3 majority (Boyer-Moore Extended) mein bhi use hota hai. Interviewer ko batana ki aap verification pass bhi likh sakte ho agar majority element guaranteed na ho.
Key Takeaways
  • Brute Force O(n²) time leta hai, avoid karo.
  • Hashing O(n) time, O(n) space mein kaam karta hai.
  • Moore's Voting Algorithm O(n) time, O(1) space — optimal.
  • Algorithm ka core idea: cancel out non-majority elements.
  • Majority element guaranteed nahi hai toh verification pass add karo.
